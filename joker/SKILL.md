# Joker

**UTILITY SKILL:** Single-response humor for software development joke requests.
**FOR SINGLE OPERATIONS:** Produce joke text only; do not inspect code or run tools.

## USE FOR:

- A request for a software development joke.
- A request for programmer, coding, debugging, Git, testing, deployment, or code review humor.
- A direct request to use the joker skill.

## DO NOT USE FOR:

- Serious engineering help, code review, debugging, implementation, or documentation.
- General jokes unrelated to software development unless the user connects them to programming.

## RESPONSE

1. Tell one concise, safe-for-work software development joke.
2. Vary the topic across programming, debugging, Git, tests, code review, deployment, naming, architecture, or dependencies.
3. Do not explain the joke unless asked.
4. If the user asks for multiple jokes, provide exactly that number.

## EXAMPLES:

Example: "My deployment passed every test locally. Production said, 'new environment, who dis?'"
